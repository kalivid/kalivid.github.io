ansible命令行常用选项及参数详解。

执行shell

command模块

command模块用于在给的节点上运行系统命令，比如echo ls等。
它不会通过shell处理命令，因此不支持$HOME这样的变量，以及<,>,|,;和$等都是无效的。

参数选项
参数 说明
chdir 运行command命令前先cd到这个目录
creates 如果这个参数对应的文件存在，就不运行command
free_form 需要执行的脚本
executable 改变用来执行命令的shell，可执行文件的绝对路径。
removes 如果这个参数对应的文件存在，就运行command
command示例

shell模块

让远程主机在shell进程下执行命令，从而支持shell的特性，如管道等。与command模块几乎相同，但在执行命令的时候使用的是/bin/sh。

参数选项
参数 说明
chdir 运行command命令前先cd到这个目录
creates 如果这个参数对应的文件存在，就不运行command
free_form 需要执行的脚本
executable 改变用来执行命令的shell，可执行文件的绝对路径。
removes 如果这个参数对应的文件存在，就运行command
shell示例

raw模块

raw模块主要用于执行一些低级的，脏的SSH命令，而不是通过command模块。 raw模块只适用于下列两种场景，第一种情况是在较老的（Python 2.4和之前的版本）主机上，另一种情况是对任何没有安装Python的设备（如路由器）。 在任何其他情况下，使用shell或command模块更为合适。

参数选项
参数 说明
free_form 需要执行的脚本
executable 改变用来执行命令的shell，可执行文件的绝对路径。
raw示例

script模块

  • script模块的作用是将本地script传送到远程主机之后执行
  • 给定的脚本将通过远程节点上的shell环境进行处理
参数选项
参数 说明
creates 如果这个参数对应的文件存在，就不运行command
free_form 需要执行的脚本
removes 如果这个参数对应的文件存在，就运行command
script示例

文件管理

file模块

file模块可以帮助我们完成一些对文件的基本操作，比如，创建文件或目录、删除文件或目录、修改文件权限等

参数选项
参数 必选项 默认值 选项 注释
follow no no yes
no		 表示文件系统的链接，如果存在，应该follow
force no no yes
no		 当state=link的时候，可配合此参数强制创建链接文件，当force=yes时，表示强制创建链接文件，不过强制创建链接文件分为两种情况，情况一：当你要创建的链接文件指向的源文件并不存在时，使用此参数，可以先强制创建出链接文件。情况二：当你要创建链接文件的目录中已经存在与链接文件同名的文件时，将force设置为yes，回将同名文件覆盖为链接文件，相当于删除同名文件，创建链接文件。情况三：当你要创建链接文件的目录中已经存在与链接文件同名的文件，并且链接文件指向的源文件也不存在，这时会强制替换同名文件为链接文件。
group no None 用于指定被操作文件的属组，属组对应的组必须在远程主机中存在，否则会报错。
mode no None 用于指定被操作文件的权限，比如，如果想要将文件权限设置为”rw-r-x—“，则可以使用mode=650进行设置，或者使用mode=0650，效果也是相同的，如果你想要设置特殊权限，比如为二进制文件设置suid，则可以使用mode=4700
ownerno no None 用于指定被操作文件的属主，属主对应的用户必须在远程主机中存在，否则会报错。
path yes 必须参数，用于指定要操作的文件或目录，在之前版本的ansible中，使用dest参数或者name参数指定要操作的文件或目录，为了兼容之前的版本，使用dest或name也可以
recurse no no yes
no		 当要操作的文件为目录，将recurse设置为yes，可以递归的修改目录中文件的属性。
selevel no
serole no None
setype no None
seuser no None
src no None 当state设置为link或者hard时，表示我们想要创建一个软链或者硬链，所以，我们必须指明软链或硬链链接的哪个文件，通过src参数即可指定链接源

hard

touch
absent		 file：文件
